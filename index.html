Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • La WWW se conoce como la web y es un sistema de información, mientras que la Internet es una red de computadoras que interconecta a los dispositivos físicos de manera global.

  • ¿Cuáles son las partes de un URL?

    • - La primera parte es el protocolo a utilizar para la transferencia de datos. Se le denomina scheme.
    - La segunda parte es el subdominio de la URL como "www", "blog", etc. La más común es "www".
    - La tercera parte es el dominio de la URL. Lo que usualmente describen las personas cuando se refieren a un sitio web.
    - La cuarta parte le denominan el dominio de nivel tope. Es usualmente del país y/o compañía a la que pertenece el dominio
    - La última parte es usualmente la ubicación de la página web a cargar y se denomina path. Por ejemplo: google.com /search
    - En algunos sitios podemos encontrar query strings que son peticiones a la página web. Al pulsar el link inferior, se abrirá una búsqueda en google buscando "cute dogs"
    https://www.google.com/search?q=cute+dogs

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET se utiliza para pedir datos y contienen un cuerpo de respuesta

    HEAD se usa para pedir datos pero no contiene un cuerpo de respuesta

    POST manda datos al servidor. Por ejemplo, al llenar una encuesta con el semantic header form. Los manda sin encriptar y puede ser texto plano, imgs, etc. POST es el único método que siempre es no idempotente, pues cambia el estado del servidor.

    PUT hace lo mismo que POST pero es siempre idempotente. Llamarlo varias veces no altera el resultado ni el estdo del servidor

    PATCH es un conjunto de instrucciones, que a diferencia del PUT, no se obtiene un cuerpo de respuesta

    DELETE es una instruccion

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    POST es el método a utilizar pues el usuario y contraseña viajan en un paquete y puede cifrarse. En cambio, un GET pondría el texto enviado después de la URL, dejándolo vulnerable.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    Es un GET, utilizando las primeras partes de la URL para direccionar el pedido y obtener el "documento". Obtenemos un cuerpo de respuesta (el contenido, comportamiento y visuales de la página)

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • 200 quiere decir que se logró resolver la respuesta con contenido en la respuesta, sin embargo puede significar algo malo cuando usamos PUT o DELETE.
    En los casos de PUT y DELETE usualmente queremos un 204 porque quiere decir que el request no regresó contenido.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No necesariamente. Podemos registrar la respuesta de una página existente a un error 404 por cuestiones de seguridad o simplemente por migración. También depende si el usuario pierde conexión con el sitio web o si los servidores se caen.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • De nuevo, no necesariamente. Un error 500 es un error genérico. De la misma manera que podemos ver una señal en el coche que algo está fallando en el motor, pero no sabemos con especificidad. Un error 500 sólo indica que no se pude completar el request del usuario. Algunos ejemplos: La base de datos se congeló, el cache del explorador está generando errores en las peticiones, las cookies del navegador están generando problemas, etc.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Son atributos que ya no deberían de usarse en proyectos nuevos y deberán ser sustituidos si existen en algún proyecto. Algunos ejemplos de elementos y atributos se presentan debajo.

    Elementos sustituidos por CSS

    1. plaintext, pues existe "text-decoration"
    2. strike, pues existe"text-decoration"
    3. menu, pues usamos unordered list "ul"
    4. center, pues usamos "text-align:center"

    Atributos sustituidos en HTML5

    1. "version" en html (innecesario)
    2. "profile" en head (innecesario)
    3. "target" en link (innecesario)
    4. "nohref" en area (simplemente omitir href implica un nohref)
